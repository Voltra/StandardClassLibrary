11 
12 Here is the namespace hierarchy:
13 
14 * [scl](./include/scl/scl.hpp) - The englobing namespace
15  * [concepts](./include/scl/concepts/concepts.hpp) - A concept-like library based around `static_assert` (mostly implemented as defined in [cppreference](https://en.cppreference.com/w/cpp/header/concepts))
16  * [tools](./include/scl/tools/tools.hpp) - General purpose utility functions
17  * [make](./include/scl/tools/make/make.hpp) - Create objects/pointers through factories
18  * [iostream](./include/scl/tools/iostream/iostream.hpp) - IOstream manipulators and utils
19  * [log](./include/scl/tool/iostream/log/log.hpp) - Logging utils
20  * [prefixed](./include/scl/tool/iostream/log/prefixed.h) - Log w/ a prefix
21  * [surrounded](./include/scl/tool/iostream/log/surrounded.h) - Log w/ a prefix & a suffix
22  * [wrap](./include/scl/tool/iostream/log/log.hpp) - An alias for [scl::tools::iostream::log::surrounded](./include/scl/tool/iostream/log/surrounded.h)
23  * [meta](#scl-tools-meta) - Template Meta Programming utils (such as `void_t`, `enable_if_t` and `constexpr` functions like `is_same` or even handy stuff like `constexpr_assert`)
24  * [console](./include/scl/tool/iostream/log/log.hpp) - An alias for [scl::tools::iostream::log](./include/scl/tool/iostream/log/log.hpp)
25  * [cli](./include/scl/tool/iostream/log/log.hpp) - An alias for [scl::console](./include/scl/tool/iostream/log/log.hpp)
26  * [exceptions](./include/scl/exceptions/exceptions.hpp) - Exception types used in the library
27  * [utils](./include/scl/utils/utils.hpp) - Utility classes (and rare functions), eg. `Any` or `Opiontal` or even`toString`
28  * [stream](./include/scl/stream/stream.hpp) - The stream API exposing stream creators, operations and terminators
29  * [details](./include/scl/stream/details/details.hpp) - Implementation details for streams (including the base iterator types)
30  * [creators](./include/scl/stream/creators/creators.hpp) - Grouping stream creation functions
31  * [operators](./include/scl/stream/operators/operators.hpp) - Grouping intermediate stream operation functions
32  * [terminators](./include/scl/stream/terminators/terminators.hpp) - Grouping end of stream operation functions

36 Here are some [macros](./include/scl/macros.h) and their meaning:
37 
38 * Code base maros
39  * `realConst(type)` - Gives a `const` version of the given type (equivalent to `const type const`, useful for pointer)
40  * `stringLiteral` - The actual string literal type (so you don't have to learn it by heart)
41  * `assert_concept` - Compile time assertions of concepts outside of a block of code (must be simple concepts, not aggregates (e.g. `Copyable ` vs `Invocable`))
42  * `static_require` - Require concepts without giving an additional error message
43  * `META` - The fully qualified namespace path to the `meta` namespace (actually is `scl::tools::meta`)
44 * Flags
45  * `SCL_CPP14` - Define this flag to use the C++14 add-ons (e.g. `is_final_t`)
46  * `SCL_CPP17` - Define this flag to use the C++17 add-ons (e.g. `is_swappable_t`)
47  * `SCL_CONCEPTS_REQUIRE_IS_VOID` - Define this flag to replace `constexpr bool require()` with `constexpr void require()`
48  * `SCL_USE_IOSTREAM_EXTEND` - Define this flag if you wish to use the general purpose `operator<<` for `ostream` that uses `std::to_string` and `scl::utils::toString` for conversions
49  * `SCL_ENABLE_DEPRECATED_TYPEMODS` - Define this flag if you wish to define the deprecated typemods (e.g. `result_of_t`)
50  * `SCL_ENABLE_DEPRECATED_TYPECHECKS` - Define this flag if you wish to define the deprecated typechecks (e.g. `is_pod_t`)
